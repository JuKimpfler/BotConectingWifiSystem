1. Save Config Button Feedback (Bug #1)
   ─────────────────────────────────────
   Problem: Der "Save Config" Button schien sich aufzuhängen, da kein Feedback
            nach erfolgreichem Speichern angezeigt wurde.

   Ursache: Der ACK-Handler für Settings hat den Timeout korrekt gelöscht, aber
            keine erfolgreiche Bestätigung an den Nutzer ausgegeben.

   Lösung: ACK-Handler wurde erweitert um Erfolgs-/Fehlermeldungen anzuzeigen:
           - Bei ACK_OK (status=0): "Settings saved successfully"
           - Bei Fehler: "Settings save failed (status X)"

   Datei: ESP_Hub/ui/src/main.js (Zeilen 498-512)

2. WebSocket Multi-Client Problem (Bug #2)
   ────────────────────────────────────────
   Problem: Bei Neuladen der Website wurde der User als neuer/zweiter/dritter
            Client angemeldet.

   Status: BEREITS BEHOBEN im Code

   Lösung: Der WebSocket-Client schließt beim Reconnect zuerst die alte
           Verbindung explizit, bevor eine neue aufgebaut wird.
           Dies verhindert, dass der Server mehrere Verbindungen vom
           selben Client gleichzeitig zählt.

   Datei: ESP_Hub/ui/src/ws.js (Zeilen 25-29)
   Code:
   ```javascript
   if (socket) {
     try { socket.close() } catch (_) { /* ignore */ }
     socket = null
   }
   ```

3. Manual Mode Controls (Bug #3)
   ──────────────────────────────
   Problem: Texteingaben und Switches im Manual Modus wurden angeblich nicht
            versendet.

   Status: BEREITS FUNKTIONSFÄHIG

   Analyse:
   - Switches (SW1-3) haben Change-Listener die sofort senden (Zeile 407-411)
   - Speed-Input hat Change-Listener der sofort sendet (Zeile 414-416)
   - Alle Manual-Controls funktionieren korrekt

   Hinweis: Der Speed-Wert wird nur beim Joystick-Gebrauch als Maximum
            verwendet. Änderungen am Speed-Input triggern einen sofortigen
            Send mit den aktuellen Joystick-Werten.

4. Zyklisches Senden im Manual Mode (Bug #4)
   ─────────────────────────────────────────
   Problem: Manual-Steuerung sollte zyklisch gesendet werden (wie Joystick).

   Status: BEREITS IMPLEMENTIERT

   Lösung: Der Joystick sendet automatisch alle 50ms (JOYSTICK_SEND_MS) die
           aktuellen Werte, solange er aktiv ist.
           - Start: Bei mousedown/touchstart
           - Zyklisch: Interval sendet kontinuierlich Werte
           - Stop: Bei mouseup/touchend wird Interval gestoppt

   Datei: ESP_Hub/ui/src/main.js (Zeilen 376-390)

   Bemerkung: Dies ersetzt die alten Pfeiltasten durch einen vollwertigen
              Joystick, der präzisere Steuerung ermöglicht.

5. Angle-Wert Bug (Bug #5)
   ──────────────────────
   Problem: Angle wurde immer als 45° gesendet, nicht als eingegebener Wert.

   Status: BEHOBEN durch Joystick-Implementation

   Lösung: Der Joystick berechnet den Angle dynamisch basierend auf der
           Position des Thumbs:
           - 0° = oben (North)
           - 90° = rechts (East)
           - 180°/-180° = unten (South)
           - -90° = links (West)

           Berechnung: Math.atan2(dx, -dy) * (180 / Math.PI)

   Datei: ESP_Hub/ui/src/main.js (Zeile 335)

6. Globale SAT1/SAT2 Auswahl (Bug #6)
   ───────────────────────────────────
   Problem: Es gab keine allgemeine Auswahlmöglichkeit zum Wechsel zwischen
            SAT1 und SAT2 außerhalb des Manual Modes.

   Status: BEREITS IMPLEMENTIERT

   Lösung: Globale Target-Selektion wurde im Header hinzugefügt:
           - Buttons "SAT 1" und "SAT 2" im Header
           - Sichtbar auf allen Tabs
           - Aktiver Button wird hervorgehoben
           - targetRole wird global für alle Commands verwendet

   Dateien:
   - ESP_Hub/ui/index.html (Zeilen 14-17)
   - ESP_Hub/ui/src/main.js (Zeilen 229-236)
